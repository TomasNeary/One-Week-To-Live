One Week To Live - External Mods Reference
==========================================

Purpose
-------

This file documents the non-OWTL mods bundled in this repository and how each one may help implement the One Week To Live gameplay plan.

These mods are reference material unless explicitly selected for integration. Do not edit bundled third-party mods directly. Build One Week To Live features under OWTL-prefixed folders or a new OWTL module.

Existing OWTL folders are not treated as external mods:

- Bows/OWTL_Bows
- Buildings/OWTL_Traps
- Horde_Night/OWTL_Horde_Night


High-Value References
---------------------

These are the most directly useful external mods for version 1:

- Horde_Night/Expanded Helicopter Events
- Buildings/PlayerTraps
- Buildings/Nolan's Traps Updates
- Buildings/Improved Build Menu [Build 41.51+] (no itemtweak)
- Bows/Remastered Kitsune's Crossbow Mod

1. Bows/Remastered Kitsune's Crossbow Mod

Usefulness: High.

Helpful for:

- Early-game ranged weapon support.
- Bows/crossbows as a low-noise alternative to firearms.
- Bolt/ammunition crafting and recovery.
- Weapon attachment patterns.
- Reload timed actions.
- Zombie-attached weapon spawning.

Observed files:

- media/scripts/module_KCMweapons_Items.txt
- media/scripts/module_KCMweapons_Recipe.txt
- media/lua/client/KCMCrossbowClient.lua
- media/lua/client/TimedActions/ISKCMReloadCrossbowAction.lua
- media/lua/client/Definitions/AttachedCrossbowDefinitions.lua
- media/lua/server/items/ProceduralDistributions_KCMnew.lua

Observed patterns:

- Adds hand crossbow, compound bows, handmade bows, bolts, broken bolts, bolt heads, shafts, fletching, and bow attachments.
- Uses custom reload timed action logic.
- Uses OnWeaponSwingHitPoint and OnWeaponHitCharacter hooks for crossbow behavior.
- Stores bolt condition in weapon mod data.
- Uses low sound radius values, useful for early horde prep weapons.

Plan relevance:

- Strong basis for OWTL early-game ranged scaling.
- Useful source for bow/crossbow item tuning, reload UX, ammo recipes, and projectile recovery.

6. Buildings/PlayerTraps

Usefulness: High.

Helpful for:

- Trap placement.
- Player trap damage.
- Square mod-data state.
- Client/server trap synchronization.
- Consumable and repairable trap states.

Observed files:

- media/scripts/traps.txt
- media/lua/client/traps.lua
- media/lua/server/TrapServer.lua
- media/lua/server/Loading.lua

Observed patterns:

- Defines BearTrap, BearTrapClosed, SpikeTrap, SpikeTrapClosed, PropaneTrap, and Nothing.
- Uses recipe OnCreate:SetTrapDown to place traps.
- Stores isTrapSet on square mod data.
- Stores isSet on item/world item mod data.
- Uses sendClientCommand with module Trap and command SetTrap to sync server state.
- Player update checks current square and applies trap effects.
- Trap effects include fractures, deep wounds, burns, explosions, and replacement with closed/worn-out trap items.

Plan relevance:

- Direct basis for OWTL spiked pits, spiked log barricades, player-damaging traps, repairable trap states, and consumable explosive traps.
- Needs adaptation for zombies, not only players.
